A MERN-stack application with a REST API that interfaces with a database that stores data about different movies.

Endpoint HTTP Method Description Request Body Response Body
/movies GET Return a list of ALL movies to the user. None. A JSON Object containing data about all of the movies.
/movies/{title} GET Return data (description, genre, director, image URL, whether itâ€™s featured or not) about a single movie by title to the user. None. A JSON Object containing data about a single movie. Example: 
    {
        id: 1,
        title: "Blade Runner 2049",
        director: {
            name: "Denis Villeneuve",
            bio: "Denis Villeneuve is a French Canadian film director, writer, and producer.",

        },
        genre: "Science Fiction",

/genres/{name} GET Return data about a genre by name. None. Text containing a short description of a given genre.
/directors/{name} GET Return data about a director (bio, birth year, death year) by name. None. A JSON Object containing data for a single director: Example: 
    {
        name: "Denis Villeneuve",
        bio: "Denis Villeneuve is a French Canadian film director, writer, and producer.",
        birth: "1967",
        death: "N/A",
    }
/register GET Return a form to allow new users to register None. A short form for the user to fill out. User is required to input an email address, date of birth, and display name. They are assigned an id.
/register POST Submit form to allow new users to register. A JSON Object containing data input by the user: Example: 
    {
        email: 'user@email.com',
        display: 'xXUser_NameXx',
        DOB: '1991-03-25',
    }
Text letting new user know their account has been successfully created.
/users/{userId} GET Return a form to allow users to update their user info. None Short form where users can change their email address, display name, or date of birth.
/users/{userId} PUT Allow users to update their user info. A JSON Object containing data input by the user: Example: 
    {
        email: 'user@newemail.com',
        display: 'New_User_Name',
        DOB: '2001-06-22',
    }
Text letting the user know that changes to their user info were made successfully.

/users/{userId}/favorites/add POST Allow users to add a movie to their list of favorites. None. Text letting user know movie was added to favorites
/users/{userId}/favorites/remove DELETE Allow users to remove a movie from their list of favorites. None. Text including message letting user know deletion was successful, and link back to their favorites page.
/users/{userId}/deregister DELETE Allow existing users to deregister. None. Text including message letting user know deletion was successful, and link back to home page.
